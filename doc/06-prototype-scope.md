# 06. 프로토타입 범위 & 로드맵

> 1인 개발. 핵심은 "재미가 검증되는 가장 작은 빌드"를 빠르게 만드는 것.
> **방향 전환 (2026-07-03): 전술 레이어 우선.** 아래 6.1 참고.

## 6.1 프로토타입의 목적 (무엇을 증명하는가)

이 게임의 정체성은 **연속 줌**이지만, **훅(kick)은 전술 파트에 있다.**
Hex 자동전투 시뮬은 흔해서 그 자체로는 끌어당기기 어렵고, "직접 개입해 눈앞에서 보상받는
시각적 재미"는 대부분 전술에 몰려 있다. 그래서 순서를 뒤집는다:

> **전술 레이어 하나를 독립적으로 먼저 완성**하고, **그것을 중심으로 전략·경영을 붙여 나간다.**

**격리 설계가 이걸 가능하게 한다** ([04 4.6](./04-tactics-layer.md) 레이어 격리·최소 인터페이스):
전술은 떼어내 **단독 실행**할 수 있다.
- **입력** = 교전 스냅샷(슬라이스에선 **하드코딩**: 지형 + 양측 부대 편성).
- **출력** = **승/패 + 승리의 정도**(콘솔/토스트로 확인).
- 전략이 함께 돌지 않으므로 **이중 시계·`TIME_RATIO`·전략 상황 알림은 슬라이스에서 불필요** → 복잡도 대폭 감소.

증명할 질문:

> **"전략 없이도, 전술 개입 그 자체가 손맛과 보상을 주는가?"**
> **"덩어리 표현 + 접전 폭·치명율 전투 + 액티브 포즈 개입이 '한 판 더' 하고 싶게 만드나?"**

이게 통과하면 그 전술을 **콘텐츠로 삼아** 전략(Hex 지휘)과 경영(턴제)을 위·아래로 확장한다.

## 6.2 전술 수직 슬라이스 v0 (첫 빌드 — 구성 확정 2026-07-03)

**목표:** 한 개의 전술 교전이 **혼자서** 재미있는지 검증. **착수 전 스펙 먼저 확정** → 상세 스펙 [04 4.8](./04-tactics-layer.md).

### ✅ 슬라이스에 포함
- **연속 공간 전장 1개** ([04 4.2.1](./04-tactics-layer.md)) — Hex 아님. 지형 = **랜덤으로 3종 생성해 템플릿으로 저장 후 재활용**, 평지·언덕 요소 포함.
- **1:1 교전(대칭)** — 각 부대 = **방패 1000·창 600·궁 400·기병 200 + 장수 1**. (기병 포함으로 변경 → 돌파·회전 로직 필요)
- **덩어리 렌더** — 앵커 + 대형 슬롯 + 저주파 노이즈, **facing 연속 각도**, 최소두께 10·폭 확산, **사상자 좌우→중앙 수렴**.
- **교전 자동 계산** ([03 3.6.2](./03-strategy-layer.md)) — 접전 폭 병목 + 공속 + **치명율(전사/부상)** + 사기.
- **액티브 포즈** — 개입 순간 정지, 명령 입력, 재개.
- **전술 개입 2종** — **돌격 + 방어전념**(전술명령). 상세 [04 4.8](./04-tactics-layer.md).
- **장수** — 출진/휴식, 근접 교전, **부대군기 명령반경**(통솔 30배/100) 표시.
- **결과 출력** — 승/패 + **정도**(잔존 전력·사기 격차 기반) 표시.
- **더미 AI** — 고정 대형 또는 단순 돌격(재미 판단엔 이 정도면 충분).

### ❌ 슬라이스에서 제외 (전략·경영 붙일 때 / v0.5 이후)
- Hex 전략 전장, 부대 이동/편성, 이중 시계·`TIME_RATIO`, 전략 상황 알림
- 협공(2:1) 동적 구성, 전략계략, 경영 턴·자원·세이브 (기병은 **포함**으로 변경)
- 전술계략(화계 등)·간파·위보

## 6.3 단계별 로드맵 (전술 우선으로 재정렬)

### 0단계 — 기술 스파이크 (전술 렌더)
- [ ] PixiJS로 모바일 브라우저에 **연속 공간 + 덩어리 스프라이트(앵커+슬롯+노이즈)** + **핀치-줌 카메라**.
- [ ] **facing 회전** 렌더, 덩어리 밀집/산개, 사상자 시 좌우→중앙 수축 표현.
- [ ] 화면 방향/성능 기준선 확인.
- **산출물:** "연속 공간 덩어리 표현을 이 스택으로 만들 수 있는가" 판단.

### 1단계 — 전술 수직 슬라이스 (★ 최우선 정체성 검증)
- [ ] 하드코딩 교전 스냅샷(지형 + 1:1 편성) 로드.
- [ ] **교전 자동 계산**(접전 폭·공속·치명율·사기) + 덩어리 사상자 반영.
- [ ] **액티브 포즈** + **전술 개입 2종** + 장수 근접/명령반경.
- [ ] 승/패 + 정도 산출.
- [ ] **검증: 전술 개입이 혼자서 재미있나?** ← 여기서 멈추고 냉정하게 판단. (통과 못하면 정체성 재설계)

### 2단계 — 전략 레이어 (전술을 감싸기)
- [ ] **Hex 전장**, 한 타일 한 유닛, 부대 이동/선택, **전방(facing)** 전환, 측후면 보정.
- [ ] 부대 6~12개 자동 교전 → **한 교전을 확대해 1단계 전술로 진입**(스냅샷 전달) → 승/패 복귀.
- [ ] **이중 시계**(`TIME_RATIO`) + 줌 인/아웃 + 전략 상황 알림 1~2종.
- [ ] **검증: 지휘하다 파고들고 빠져나오는 연속 줌 흐름이 사나?**

### 3단계 — 경영 레이어 최소 연결
- [ ] 노드 맵 + 부대 이동(턴제), 도시·자원·부대 편성.
- [ ] 접촉 → 전투(전략/전술) → 결과 반영 루프 완성 + 자동 해결.

### 4단계 — 게임 루프 완성
- [ ] AI 1세력과 한 판 캠페인(시작~승부), 세이브/로드, 기본 HUD.
- [ ] **검증: 한 판이 끝까지 재미있는가?**

### 5단계 — 다듬기 & 확장 (선택)
- [ ] 협공(2:1)·전술계략·간파, 발동형 지형, 인재·외교, `TIME_RATIO`·알림 밸런싱.
- [ ] PWA/Capacitor 앱 배포 검토.

## 6.4 핵심 검증 지점 (Kill / Go)

각 단계 끝에서 냉정하게 묻는다. 재미가 없으면 다음으로 가지 말 것.

| 단계 | 질문 | 통과 못하면 |
|------|------|-------------|
| 0 | 연속 공간 덩어리 표현이 이 스택으로 가능한가? | 스택/표현 방식 재검토 |
| 1 | **전술 개입이 — 전략 없이도 — 재미있나?** | 정체성 재설계 (가장 중요) |
| 2 | 지휘→줌인→개입→복귀 연속 줌 흐름이 사나? | 전략↔전술 인터페이스 재설계 |
| 4 | 한 판 캠페인이 "한 판 더" 하고 싶나? | 루프/페이싱 조정 |

## 6.5 1인 개발을 위한 원칙

- **수직 슬라이스 우선** — 넓게 깔지 말고, 한 줄기를 끝까지 동작하게. (**전술 슬라이스가 그 첫 줄기**)
- **에셋은 임시로** — 도형·무료 에셋으로 시작. 아트는 재미 검증 후.
- **밸런싱은 나중** — 먼저 "동작", 그 다음 "재미", 마지막에 "공정".
- **격리를 지킨다** — 전술은 스냅샷 입력 / 승·패 출력만. 전략을 나중에 붙여도 전술 코드가 안 흔들리게.
- **문서와 코드를 함께** — 결정이 바뀌면 이 문서들도 갱신.
