# 03. 전략 레이어 (실시간 · Hexa 전장)

> 경영맵에서 부대가 충돌하면 진입하는 **실시간 전투**의 큰 화면.
> **Hexa 타일** 전장 위에서, **부대(유닛) 단위로** 지휘한다. 표현은 간략하게.

## 3.1 전투 전체 흐름 (전략 ↔ 전술)

```
[경영(턴제)]  부대 이동 → 적과 접촉
      │
      ▼
[전략(실시간)]  Hexa 전장에 부대 배치. 시간이 흐른다(빠름).
      │   인접한 적/아군이 교전 → 지형·장수·능력치로 "자동 계산"
      │   ├─ 그대로 두면: 사기 저하/전멸로 자동 결판
      │   ├─ 전략적 개입: 재배치·전방 전환·증원·후퇴 명령 (부대 단위)
      │   └─ 확대(줌인): 특정 교전을 ──→ [전술(실시간, 느림)]
      │                                       유저가 직접 개입
      │   ↑ 전술에 몰입한 동안에도 전략은 계속 흐름(빠름)
      │     → 중요한 전략 상황은 전술 화면에 "알림"으로 공유
      ▼
[전투 종료]  생존 병력·사상자·장수 생사 → 경영 레이어에 반영
```

> 핵심: **전략의 교전은 기본적으로 자동 계산된다.** 플레이어는 (1) 전략적 명령으로 개입하거나
> (2) 줌인해 전술적으로 직접 싸운다. 줌인의 시간 규칙은 [04 문서](./04-tactics-layer.md).

## 3.2 시간 모델: 실시간 + 액티브 포즈

전투는 **실시간으로 진행된다.** 부대는 명령에 따라 이동·교전하고, 사기에 따라 무너진다.

플레이어가 **명령을 내리려 개입하는 순간 전투 전체가 정지한다 (액티브 포즈).**
- 부대 선택 → 정지. 이동/공격/전방 지정을 천천히.
- 정지 상태에서 여러 부대에 명령 **예약** → 재개 시 일제 실행.

> **전략 시계는 빠르게 흐른다.** 전술로 줌인하면 그 교전만 느린 시계로 펼쳐지고, 나머지 전장은
> 전략 시계(빠름)로 계속 돈다. 둘은 같은 전투지만 **시간 비율이 다르다.** → [04 문서 4.3](./04-tactics-layer.md)

### 3.2.1 지휘 통제 — 부대군기와 명령 반경 (1차 확정 2026-06-30)

명령은 **부대군기(部隊軍旗)를 중심으로 한 반경 안에서만** 하달된다. **군기 운용이 곧 지휘**다.
(부대군기 엔티티·장수 리젠 규칙 → [07 7.1](./07-entities-data.md))

- **부대군기는 매우 느리다** — 일반 보병 유닛의 **1/2 속도**. 전선을 밀려면 군기를 먼저 전진시켜야 한다.
- 부대군기는 **지휘 반경(command radius)** 을 가진다. 액티브 포즈로 명령할 때 **반경 내 유닛에게만** 명령이 닿는다.
- **반경 크기 = 장수 통솔(統)에 비례** (확정 2026-06-30). 통솔 높은 장수일수록 더 넓게 지휘한다.
- 군기가 느리고 반경이 제한적 → "지휘가 닿는 범위"를 어디에 두느냐가 전략 핵심. 무리한 전진은 군기 노출 위험.

**명령 시퀀스 예 — 「돌격」:**
1. 액티브 포즈 → 「돌격」 커맨드 선택. **장수가 기본 선택**된다.
2. 함께 돌격할 유닛 선택 — **반경 내 유닛만** 선택 가능.
3. **돌격 지점** 지정.
4. 입력 완료 → 장수 + 선택 유닛이 **대형을 이루고** 지점으로 돌격 개시(재개 시 일제 실행).

> ⚠️ **명령 ≠ 계략, 그리고 계략도 전략/전술로 나뉜다.** 3종(전략계략·전술계략·전술명령) 비교표와
> 전술명령 예시(돌격·기마저지·방어전념) → [04 4.5.1](./04-tactics-layer.md).

**계략(장수 특수기 — 화계·낙석·매복 등):** 장수가 **제자리에서 작전포인트를 차징**해 발동. 차징 중 피격
누적이 임계를 넘으면 취소. 발동 범위 = **지휘 반경 + 지력(智)**. 차징 중 **상대는 지력+운 판정으로 간파**
가능(특기 **간파**=주사위↑, **위보**=상대 주사위↓ — 거짓정보 없음). 상세 → [07 7.1 장수 상태 & 계략](./07-entities-data.md).

> 🟡 OPEN: 통솔→반경 환산식(계수), 군기 속도의 정확한 값, 반경 밖 유닛의 행동(자동 방어?),
> 다른 커맨드 셋(후퇴·집결·방어 등), 작전포인트 차징/취소 수치. → [04 4.7](./04-tactics-layer.md)와 묶어 확정.

## 3.3 Hexa 전장 구조

- **육각 타일(Hex) 그리드.** 지형도 Hex 타일 단위로 구성.
- **한 타일에는 한 유닛만.** 부대는 겹치지 않는다 — 위치 선점과 진형이 곧 전술.
- 표현은 **간략하게**: 부대는 토큰/카운터 + 병종 심볼 + 사기·병력 바. (개별 병사는 전술 줌에서만)
- 지형 타일(예시): 평지 / 고지(사거리·방어↑) / 숲(은폐·이동↓) / 강·다리(병목) / 도하지점.
  - 같은 지형이 **전술에서는 구체 연출/상호작용**(언덕 경사·물가·낙석)으로 확장된다. → [04 문서](./04-tactics-layer.md)

## 3.4 유닛: 6면과 전방(Facing)

- 각 부대는 **6면을 가진 Hex 유닛**이며 **전방(facing) 방향**을 가진다.
- **전면 / 측면 / 후면**에 따라 방어·사기 영향이 달라진다.

| 피격 방향 | 효과 (제안) |
|-----------|-------------|
| 전면(front) | 정상 방어. 정면 충돌. |
| 측면(flank) | 방어↓, 사기 피해↑ |
| 후면(rear) | 방어↓↓, 사기 피해↑↑ (붕괴 유발) |

- **전방 전환**에는 시간/행동 비용이 든다 → 측후방을 내주지 않게 기동하는 것이 핵심.
- 예: **아군 2부대가 적 1부대를 양쪽에서** 붙으면, 적은 한쪽에 측/후면을 내주며 빠르게 무너진다.

## 3.5 유닛(부대) 능력치 & 병종

병종 상성을 **삼각 구조**로 단순화한다.

| 병종 | 강함 | 약함 | 특징 |
|------|------|------|------|
| 창/보병 (槍) | 기병에 강함 | 궁병에 약함 | 방어·전선 유지 |
| 궁병 (弓) | 보병에 강함 | 기병에 약함 | 원거리 딜, 근접 취약 |
| 기병 (騎) | 궁병에 강함 | 창병에 약함 | 기동·측후방 돌격 |

부대 스탯(최소 집합): **체력(병력 수), 공격, 방어, 사기, 이동속도, 전방.**

## 3.6 교전 자동 계산 (시뮬레이션 코어)

이 게임은 **시뮬레이션**이다. 인접한 적대 부대는 매 틱 자동으로 교전을 계산한다.

**계산에 들어가는 변수(제안):**
- 부대 능력치(공격/방어/병력 수)
- 병종 상성 (창-궁-기)
- **전방/측후면** 관계
- **지형** 보정 (고지·숲·도하 등)
- **장수** 보정 (통솔·무력 → 공격/방어/사기 보너스, 특수기)
- **수적 관계** (협공 여부)

**결과:** 매 틱 사상자 + 사기 변동 → 사기 붕괴 시 **패주/후퇴**, 또는 전멸. 시간이 결판을 낸다.

> **사기(Morale)가 승부의 축.** 전멸보다 사기 붕괴가 먼저 온다. 측후방 협공·장수 사망·열세가
> 사기를 깎는다. 전략적/전술적 개입은 결국 "어떻게 적 사기를 빨리 무너뜨리나"로 수렴한다.

### 3.6.1 사기(Morale) 시스템 (1차 확정 2026-06-30)

사기는 **승부의 축**이며 전략·전술 양쪽에 적용된다.

**기본값**
- 각 유닛은 사기를 가진다. **범위 0~100, 시작 보통 50.**
- 장수 특기 **'고무'** 보유 시 해당 유닛 **+10**.

**증가 요인**
| 계기 | 효과 | 레이어 |
|------|------|--------|
| 커맨드 **'보급'** | 대상 유닛 +10 | 전략 |
| 상대 유닛 **후퇴 소식** 전달 | 전체 유닛 +5 | 전략 |
| 아군 **지원부대 도착 소식** | 전체 유닛 +5 | 전략 |
| **전술 교전 승리**(정도 비례) | 승리한 유닛 사기↑ (**정도만큼**) | 전술→전략 |
| **전략 계략** 성공(≠전술 계략) | 관련 유닛 사기↑ | 전략 |
| 상대 장수 HP 0 → **휴식 전환** | +10 | 전술(로컬) |
| **전술 계략 성공** | +5 | 전술(로컬) |

> **두 층위의 사기:** 위 "전술(로컬)" 항목은 그 전투 안에서만 작동해 **승/패**로 수렴하고,
> 전략 유닛의 지속 사기는 **전술 승리의 정도**와 **전략 계략 성공** 등 전략 이벤트로 바뀐다.
> **전략계략**(≠전술계략): **전략 화면 전용**, **경영에서 전투 전 선택·부여**, 사용 타이밍은 유저 결정.
> 3종 체계 표 → [04 4.5.1](./04-tactics-layer.md). (종류·간파 상세는 OPEN)

**감소 요인**
- **접전 중 시간에 따라 양측 사기가 지속 하락**(전략·전술 공통).

**효과**
- **전술 데미지 보정**: 사기가 높을수록 **상대에게 주는 피해↑**. (방어력과 무관 — **공격 측 사기**가 딜에 작용)
- **사기 ≤ 20**: 확률적으로 **명령/커맨드(돌진 등)에 반응 실패**.

**패배 → 도주 → 재편성**
- **사기 0 AND 장수 HP 0** → **부대군기 유무와 무관하게 부대 도주**(패배).
- 도주 유닛은 **대장군기로 집결**(소멸 아님) → 일정 수 + 여분 장수 시 **저사기 유닛으로 재편성**. → [07 7.1](./07-entities-data.md)
- ⚠️ **도주 ≠ 철수.** 철수는 질서 있는 후퇴, 도주는 통제 상실. ('해산' 용어는 폐기 — 유닛 소멸 아님)
- 관련 특수기: **추격**(도주 적 데미지↑) · **퇴로**(아군 도주 이탈↑, 손실↓). → [07 7.1 특수기 후보](./07-entities-data.md)

> **용어 구분:** **명령(커맨드)** = 유저가 직접 내리는 지시(돌격·보급 등). **계략** = 장수가 수행하는
> 특수기(화계·낙석·매복 등). **둘은 다르다.** ([07 7.1](./07-entities-data.md))

> 🟡 OPEN:
> - 시작 사기 편차(병종/장수별?), 접전 사기 감소율, **사기→데미지 환산식**, 반응 실패 확률 곡선
> - 사기 상한 캡(고무+보급 중첩 시 100 초과 처리)
> - **도주 vs 철수** 처리 차이 (도주 후 대장군기 재편성은 [07 7.1](./07-entities-data.md)에 정리됨)

## 3.7 전략적 개입 vs 전술적 개입

| 구분 | 전략 개입 (이 문서) | 전술 개입 ([04 문서](./04-tactics-layer.md)) |
|------|---------------------|------------------------------|
| 단위 | 부대 | 부대 내부(장수·병종·지형) |
| 행동 | 재배치·전방 전환·증원·후퇴 | 장수 운용·병종 제어·지형 활용 |
| 시간 | 전략 시계(빠름) | 전술 시계(느림) |
| 성격 | 큰 그림 조율 | 자동 계산에 직접 손을 댐 |

## 3.8 전투 종료 & 결과 반영

- 한쪽 전멸/전면 패주 또는 군주 후퇴 시 종료.
- 결과(생존 병력·사상자·장수 생사)가 **경영 레이어에 반영** → [02 문서](./02-management-layer.md).
- 자동 해결(auto-resolve)은 동일 전투 로직을 빠르게 시뮬레이션해 결과만 산출.

## 3.9 열린 질문

- Hex 이동/사거리 규칙의 정밀도(인접=1, 거리 기반 사격 등) — 단순부터.
- 전방 전환 비용(즉시 vs 시간 소요)과 측후면 보정 수치 — 밸런싱 대상.
- 한 전장에 동시 교전이 여러 곳일 때, 줌인은 한 번에 하나만? → [04 문서 4.4](./04-tactics-layer.md)에서 논의.
- 전장 크기(Hex 수)와 동시 부대 수 상한(모바일 성능).
