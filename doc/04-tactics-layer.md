# 04. 전술 레이어 (확대한 교전 안의 전투 · 실시간 · 느린 시계)

> 전략에서 **자동 계산되던 교전**을 확대하면, 그 안으로 들어가 **유저가 직접 개입**하는 화면이 펼쳐진다.
> 장수·개별 병종·지형 연출이 드러나고, 시간은 전략보다 **느리게** 흐른다.

## 4.1 핵심 아이디어: 자동 계산에 손을 넣는다

전략 레이어의 교전은 기본적으로 **자동 계산**된다(능력치·지형·장수·전방·협공 → 사상자/사기).
플레이어가 그 교전을 **확대(zoom-in)** 하면, 자동으로 흘러가던 전투가 **유저가 개입 가능한
전술 전장**으로 전개된다.

```
   축소 ◀───────────────────────────────────▶ 확대
  [전략: 자동 계산]                    [전술: 유저 개입]
 토큰/카운터, 빠른 시계         장수·개별 병종·지형 연출, 느린 시계
```

- **줌 인의 의미** = "이 교전만큼은 내가 직접 지휘해 결과를 바꾸겠다."
- 줌 인하지 않은 교전은 전략 시계 위에서 계속 자동으로 결판난다.

## 4.2 확대하면 보이는 것

전략에서 토큰 하나였던 교전이, 확대하면 살아 움직인다.

| 요소 | 전략(축소) | 전술(확대) |
|------|-----------|-----------|
| 장수 | 깃발/아이콘 | **통솔 중인 장수 본인**이 등장 (위치·상태) |
| 병종 | 부대 심볼 | **창·궁·기 개별 병사 단위**로 구분·배치 |
| 지형 | Hex 타일 보너스 | **연출·상호작용**: 언덕 경사·물가·낙석 등 |
| 전투 | 수치 자동 계산 | 병사 간 근접/사격이 눈에 보이는 형태로 |

### 4.2.1 전술 표현 모델 — 연속 공간 + 대형 덩어리 (확정 2026-07-01)

전술은 **Hex를 쓰지 않는다.** **연속 2D 공간** 위에서 대형과 기동을 표현한다.
(전략은 Hex 유지. 전술은 **격리 레이어** — 진입 시 교전 스냅샷을 받고, 전략엔 **승/패만** 되돌린다. [05](./05-tech-architecture.md) · 4.6)

- **장수**: 단일 에이전트. 자유 이동 + 개별 전투 주체(HP·일기토). → [07 7.1](./07-entities-data.md)
- **병종 그룹**(방패/창/궁/기): 각 그룹을 **하나의 엔티티로 시뮬**한다.
  - 상태 = `{대형 앵커, 대형형태(밀집/산개), 방향(연속 각도), 병력수, 사기}`
  - 개별 병사는 **앵커 주변 대형 슬롯 + 저주파 노이즈(꿈틀)** 로 **렌더만** — 독립 AI·충돌 시뮬 없음.
  - 전투 = 그룹 **전선 접촉** 판정, 피해 → 병력수↓ → 스프라이트 제거.
- **대형 깊이**: 앞 방패 → 창 → 궁, 더 뒤 기병, 중앙 **부대군기 + 호위병**. (4.5 대형 명령과 연결)
- **facing**: 연속 각도. 전략 Hex 6방향 → 전술 각도로 매핑(측/후면 판정이 더 정밀).
  방향 전환 속도 = 병종 **'회전'** 능력치 → [07 7.2](./07-entities-data.md). 대열을 잘 돌리는 게 곧 방향성 표현.
- **지형**: 혼합 타일맵(대표 지형 ≥60%) 위에 유닛을 연속 좌표로 배치·이동. → [07 7.3](./07-entities-data.md)
- **겹침 규칙 (개정 2026-07-08):**
  - **같은 진영 병종끼리는 겹침 허용.** 방패 앞·창 바로 뒤처럼 촘촘히 붙여 대형 깊이를 표현(전면 병종만 겹쳐 앞서고 뒷 병종이 붙는다).
  - **반대 진영끼리는 전면 "가장자리"만 겹침**(접전 표현, `edgeOverlap`). **그 이상 겹침·통과 불가** — 밀어내며 전선에서 막힌다. 이 접촉면이 곧 **접전 폭**([03 3.6.2](./03-strategy-layer.md))의 근거.
  - **예외 = 돌파:** **기병 돌격(charge)** 은 반대 진영을 통과(뒷열 타격). **장수**도 이동속도가 느려지지만 돌파 가능(설계; 이동 미구현).
  - (구 규칙: "병종끼리 무조건 미겹침" → 같은 진영 겹침 허용으로 개정.)
- **대열 형성·사상자 표현 (확정 2026-07-03):**
  - **최소 두께 10, 최대한 넓게 확산.** 폭 = 병력 ÷ 두께 → **1000명이 500명보다 2배 넓게** 펼쳐진다.
  - **지형이 폭을 제한**하면 다 못 펼쳐 두께가 늘어난다(폭 상한 → 두께 = 병력 ÷ 실효 폭). → 병목 지형에서 유효 전면↓([03 3.6.2](./03-strategy-layer.md)).
  - **사상자 = 좌우(양끝)부터 줄고 중앙으로 수렴.** 어디서 죽든 덩어리는 가운데로 모인다(대열 구멍은 표현하지 않음). → 병력↓이면 폭↓ → 유효 전면·공격 횟수도 자연 감소.

> **왜 덩어리인가**: N명 개별 시뮬 대신 **그룹 단위 시뮬 + 시각적 스캐터**로 연산을 줄이면서
> 대형 깊이·산개/밀집·"살아 움직이는" 연출을 얻는다. 모바일·1인 개발에 현실적(Total War류 표현).

> 🟡 OPEN: 대형 슬롯 보간(boids-lite vs 고정 슬롯), 전선 접촉 판정 해상도,
> 병력수↔스프라이트 수 매핑(1:1 vs 축약). (그룹 겹침 = **미겹침**으로 확정)

## 4.3 시간 모델: 전략(빠름) vs 전술(느림) — 시계 비율

핵심 규칙. 두 레이어의 시간은 **함께 흐르지만 같지 않다.**

- **전략 시계는 빠르게**, **전술 시계는 느리게** 흐른다.
- 비율은 **튜닝 값**이다. 예: 전술 1초 동안 전략은 N초 진행 (가칭 `TIME_RATIO`, 예 N≈3~5).
- 줌 인하면 → 그 교전만 전술 시계(느림)로 상세 전개. **나머지 전장은 전략 시계(빠름)로 계속 자동 진행.**
- **액티브 포즈로 멈추면 양쪽 모두 정지.** (정지 중 줌을 오가며 명령 예약 가능)

```
플레이어 체감 시간 ──────────────────────────────▶
  전술(줌인한 교전):   천천히, 한 수 한 수 개입 가능
  전략(나머지 전장):   그 사이 빠르게 진행 → 상황이 바뀐다
```

> **설계 의도 = "주의의 안개(fog of attention)".** 한 곳을 깊게 보는 동안 다른 곳에서 일이 벌어진다.
> 시간 비율이 그 대가를 만든다 — 전술에 오래 머물수록 전략 상황은 더 많이 변해 있다.

### 왜 시계를 분리하나 (공유하지 않는 이유)
- **공유(1:1)하면:** 상세 전술 개입을 하는 사이 전장이 거의 멈춘 듯 보여 긴장감이 없다.
- **분리하면:** 전술 개입은 충분히 느려 "둘 만하고", 전략은 빠르게 흘러 "방치의 대가"가 생긴다.
- 이 비대칭이 **선택의 무게**(어디에 개입할까)를 만든다.

## 4.4 전략 상황 공유 (전술 중 알림)

전술에 몰입한 동안 플레이어가 미처 못 본 **전략적 상황이 전술 화면으로 공유**된다.

- 형태: 화면 가장자리 **알림/토스트**, 미니맵 점멸, 경고음(선택).
- 예시:
  - "아군 **A 부대 사기 저하 → 후퇴 중**"
  - "적 증원 부대 접근 (3 Hex)"
  - "B 거점 측면 붕괴 위기"
- 플레이어의 선택지:
  1. **무시하고** 현재 전술 교전에 집중 (이 교전을 빨리 끝낸다)
  2. **액티브 포즈** → 상황 파악 후 명령 예약
  3. **줌 아웃**해 전략적으로 대응 (다른 교전으로 줌인 전환 포함)

> 알림은 "전지적 시야"를 주지 않는다. **무엇을, 언제 알려줄지**가 곧 난이도/긴장 설계. (→ 4.7 OPEN)

## 4.5 전술 명령 (전략과 다른 명령어)

확대 상태에서는 전략의 부대 단위 명령과 **다른, 더 세밀한 명령어**를 쓴다. (방향성 제안)

| 분류 | 예시 명령 (제안) | 설명 |
|------|------------------|------|
| 장수 운용 | 장수 돌격 / 후방 보호 / 특수기 발동 | 장수 본인의 위치·행동 지시 |
| 병종 제어 | 궁병 일제사격 / 창병 방진 / 기병 측면 우회 | 부대 내부 병종별 개별 지시 |
| 지형 활용 | 고지 점령 / 도하 / **낙석 트리거** / 매복 | 전술 지형과의 상호작용 |
| 대형 미세조정 | 산개 / 밀집 / 방향 전환 | 전략의 큰 진형보다 정밀한 배치 |

> **전략 = "어디로 가서 누구와 싸워라"**, **전술 = "그 싸움을 어떻게 이겨라."**
> 같은 액티브 포즈 위에서 명령의 **해상도**가 다르다.

### 4.5.1 명령·계략 체계 — 3종 구분 (확정 2026-07-01)

플레이어가 개입하는 수단은 **세 갈래**다. (우리가 앞서 상세화한 "계략"[07 7.1]은 곧 **전술계략**)

| 구분 | **전략계략** | **전술계략** | **전술명령** |
|------|-------------|-------------|-------------|
| 사용 화면 | **전략 화면만** | 전술(장수 중심) | 전술 |
| 획득/출처 | **경영에서 전투 전 선택·부여** | 장수 보유 → 작전P 차징 | **부대군기 보유** 시 가능 |
| 발동 | 사용자가 **타이밍 결정** | 작전P **풀 차징** 완료 | 명령 후 대상이 **위치/조건 부합** 시 실행 |
| 사전 확정 | 전투 전 선택 | **차징 전** 종류·타겟 모두 확정 | 병종 + 집결/타겟 위치 지정 |
| 장수 | (부여 주체) | **필수**(장수 중심) | 있으면 자동 편성, **없어도 발동 무관** |
| 범위 | — | 부대군기 명령반경 **+ 지력** | 부대군기 명령반경 |
| 간파 | (OPEN) | 차징 시 상대 **지략 주사위**로 간파. **간파/위보 특기**가 주사위 ±보정 | 없음 |
| 예 | **후송·의무반**(부상병 회복) 등 (그 외 OPEN) | 화계·낙석·매복 | **이동/배치**(기본)·돌격·기마저지·방어전념 |

> 상세: 전술계략 [07 7.1](./07-entities-data.md) · 명령반경 [03 3.2.1](./03-strategy-layer.md) · 전략계략 획득 [02 2.5](./02-management-layer.md).
> **부상병 회복은 전략 화면 전용** — '후송'·'의무반' 같은 **전략계략**으로 처리한다([07 7.2 부상병](./07-entities-data.md)). 전술 교전 중엔 회복 없음.

### 4.5.2 전술명령 예시 (확정 방향)

명령은 **부대군기 명령반경** 안에서 하달되고, 대상 병종/장수가 지정 위치·조건에 부합할 때 실행된다.

- **이동/배치 (기본 명령, 확정 방향 2026-07-03)**: 이동명령 선택 → **액티브 포즈(실시간 정지)** →
  **각 병종별로 이동할 위치를 지정** → 재개 시 각 덩어리가 지정 위치로 이동(이동력·회전 규칙 적용).
  - 이것이 병종 위치를 잡는 **가장 기본 조작**이다. **전투 개시의 "장기식 배치"([4.8](#48-전술-슬라이스-v0-스펙-착수-전-확정-대상--2026-07-03))도 이 명령의 특수 케이스**(개시 시 위치 지정).
  - **방패병 방어위치 지정 = 동일 메커니즘.** 방패병이 **버티며 방어할 위치**를 같은 위치 지정으로 잡는다(아래 방어전념과 결합).
- **돌격**: 돌격할 **병종 선택** + **타겟 위치 지정** → 선택 병종과 장수(있으면)가 **지정 위치에 집결 후 타겟으로 돌격**. (이동/배치 위에 '돌격 의도'를 얹은 형태)
- **기마저지**: 우회 돌격하는 **상대 기병에 집중 타격**(창병·궁병만 가능). 병종 지정 + 집결 위치 → 그 위치에서
  **기병만 상대로 방어**하며 더 높은 데미지.
- **방어전념**: 지정 병종의 **가하는 데미지 0 · 방어력 2배**(방어에 전념). **궁병·기병 불가.** 위치는 **이동/배치(방어위치 지정)** 로 잡는다.

### 4.5.3 이동·회전 규칙 (확정 방향 2026-07-03)

실시간이므로 이동/회전은 **속도**로 흐른다.

- **이동속도 = 이동력**, **회전속도 = 회전 능력치**([07 7.2](./07-entities-data.md)).
- **회전 = 기본 제자리 회전**(방향만 틀기), 대열 유지.
- **덩어리 이동 = 모임 → 이동 → 도착지 펼침 (확정 2026-07-03).** 이동할 땐 현실적으로 **모여서(응집)** 움직이고, **목적지에서 대열로 펼친다.**
  - ∴ **이동 중(모임 상태)은 전개 전이라 전투 효율↓**(전열 폭 좁음) → 이동 중 접촉은 불리. 좋은 위치 선점의 가치.
- **기병 예외 — 선회(arc turn):**
  - **정지 상태 → 제자리 회전 가능**(느림, 회전 D). 단 **멈추면 속도 보정을 잃어 약해진다**(기병의 힘은 기동에 있음).
  - **이동 중 → 제자리 회전 불가, 크게 선회**로 각도를 조금씩 튼다.
  - **선회는 히트앤런 장치:** 치고 빠져야 하고, **돌파 후 재공격하려면 선회로 돌아 나와 다시 charge**해야 한다.
  - **선회 반경 ≈ 장수 유닛 크기의 20배**(기본, 튜닝 대상). → 우회·재돌격이 굼뜬 이유.

**명령반경과 이탈**
- **부대군기 명령반경은 제법 넓다.**
- **모든 병종은 반경을 벗어날 수 있다.** 단 **반경 밖이면 명령 하달 시 선택 불가**(재선택 불가 → 통제 상실 위험).
- **목표 지점은 반경과 무관.** 반경 안에서 선택해 **반경 밖 지점으로 보낼 수 있다**(멀리 보내면 그 뒤 재명령 불가). ← 개입의 리스크.

**기병 이동·돌격 (확정 방향 2026-07-03)**
- **이동과 공격을 동시에** 한다(스쳐지나가며 타격).
- **가속:** 이동력은 높지만 **최대속도 도달에 시간이 걸린다(기본 2초, 튜닝 대상).**
- **최대속도 도달 시 공격·방어 추가 보정 → 공격 A가 S로.**(방어도 추가 보정, 수치 OPEN)
- ∴ **런웨이가 필요한 모멘텀 무기.** **창병 저지(A)로 감속당하면 최대속도·charge 보너스가 안 뜬다**(상성표 없는 카운터 재확인). **언덕 내리막**은 가속을 도와 charge를 쉽게 터뜨린다([4.8 언덕 효과](#48-전술-슬라이스-v0-스펙-착수-전-확정-대상--2026-07-03)).

## 4.6 줌 인/아웃 & 교전 전환 — 레이어 격리 & 최소 인터페이스 (확정 2026-07-01)

**전술은 전략과 다른, 격리된 레이어다.** 두 레이어를 상태로 강하게 엮지 않는다.
시간만 `TIME_RATIO`로 비례해 함께 흐르고([4.3](#43-시간-모델-전략빠름-vs-전술느림--시계-비율)), 전략 상황은 알림으로만 전해진다([4.4](#44-전략-상황-공유-전술-중-알림)).

- **동시 상세 개입 교전 = 한 번에 하나**(권장). 단, 그 교전이 **1:1이라는 뜻은 아니다**(아래 협공).
- **진입(전략→전술):** 전략에서 결정된 **교전 스냅샷**(참가 유닛·병종·병력·사기·장수·지형)을 넘겨
  전술의 **초기 상태**로 삼는다.
- **협공(2:1 등):** 전술은 1:1만이 아니다. **협공**도 벌어지며, 참가 부대는 **전략 Hex의 인접 방향에 맞춰**
  그 방향에서 전술 전장에 진입한다(측·후면 각도로 반영 → [03 3.4](./03-strategy-layer.md)).
- **진행 중 동적 변화:** 전술 진행 중에도 전략 실시간은 계속 흐른다. 그래서 **1:1 대치가 전략 상황 변화로
  2:1이 되거나** 그 반대가 될 수 있다(인접 부대 합류/이탈). → **참가 구성은 전략에 실시간 연동**된다.
- **복귀(전술→전략):** 전술이 전략에 되돌리는 것은 **유닛의 승/패 + 승리의 정도**뿐.
  세부 사상자·배치·좌표는 되돌리지 않는다(양방향 위치 변환 불필요). **정도**는 전략 유닛 **사기**에 반영
  → [03 3.6.1](./03-strategy-layer.md).
- 줌인 대상 변경·결판 전 줌아웃 → 그 교전은 다시 **전략 자동 계산**으로 진행, 결과도 승/패(+정도).

> ✅ 최소 인터페이스(**구성 = 전략 실시간 연동 / 결과 = 승패+정도**)라 복잡한 좌표 변환 부담이 없다.
> 🟡 OPEN: 승리 '정도'의 단계·환산(잔존 전력·사기 격차 → 대승/승/신승?), 협공 시 전술 배치 규칙 상세.

## 4.7 세부 컨셉에서 정할 것 (OPEN — 함께 결정)

1. **시계 비율(`TIME_RATIO`)** — 전술:전략 = 1:N의 N 값. 체감·밸런스 핵심.
2. ~~전술 진입 단위~~ → **협공 포함(2:1 등), Hex 인접 방향대로 진입 + 진행 중 전략 실시간 연동으로 구성 변동.** (확정 2026-07-01, → 4.6)
3. **전술 명령의 최종 셋** — 4.5 중 채택/추가.
4. **지형 상호작용 범위** — 낙석 같은 발동형 지형을 어디까지(재미 vs 구현비용).
5. **장수 역할** — 버프형인가, 일기토 같은 개별 전투 주체인가.
6. **알림 정책** — 어떤 전략 이벤트를, 어느 시점에, 얼마나 보여줄지(주의의 안개 난이도).
7. **줌 전환 중 시간** — 전환 순간 자동 정지할지, 완전 실시간 유지할지.

## 4.8 전술 슬라이스 v0 스펙 (착수 전 확정 대상 — 2026-07-03)

개발 순서를 **전술 우선**으로 잡았다([06 6.1](./06-prototype-scope.md)). v0는 **격리된 전술 교전 하나**를
단독 실행해 "전략 없이도 재미있나"를 검증한다. **입력 = 하드코딩 스냅샷 / 출력 = 승·패+정도.**

### 범위 (확정 2026-07-03, 편성·지형 갱신)
- **1:1 교전.** 아군 1부대 vs 적 1부대, **양측 동일 편성(대칭).**
- **편성 = 방패 1000 / 창 600 / 궁 400 / 기병 200** (부대 합 2200) **+ 장수 1**. → **기병 재포함**(앞선 '기병 제외' 뒤집음).
- **지형 = 랜덤으로 3종을 생성해 템플릿으로 저장**, 그 **3 템플릿을 재활용**(매 판 새로 생성이 아니라 만들어둔 셋에서 사용). 평지·언덕 등 요소 포함.
- **전술 개입 = 이동/배치(기본) + 돌격 + 방어전념**(전술명령). [4.5.2](#452-전술명령-예시-확정-방향)
  - **이동/배치**: 액티브 포즈 → 병종별 목표 위치 지정 → 재개 시 이동. 방패 방어위치도 동일 메커니즘.
- ⚠️ **기병 포함 → v0 전투 계산에 돌파(전열 우회)·회전(방향전환 느림) 로직이 필요**해진다([03 3.6.2](./03-strategy-layer.md)).

### 초기 배치 (확정 방향 2026-07-03 — "장기처럼 대충")
정밀 배치가 아니라 **장기(將棋)식 개략 배치**. 양측 대칭. (= **이동/배치 명령의 전투 개시 케이스** → [4.5.2](#452-전술명령-예시-확정-방향))
- **방패 1000** → 2등분(≈500/500) **좌·우 전방**.
- **창 600** → 2등분(≈300/300) **방패 바로 뒤 좌·우**.
- **궁 400** → **중앙**.
- **기병 200** → **궁 뒤(후방)**.
- **부대군기 + 장수** → 중앙 후방. ([4.2.1](#421-전술-표현-모델--연속-공간--대형-덩어리-확정-2026-07-01))

### 전투 루프 (확정 방향 2026-07-03)
1. 접근 → **전선 접촉**(덩어리 미겹침, 접촉면 = 접전 폭).
2. 매 틱 **자동 계산**([03 3.6.2](./03-strategy-layer.md)): 접전 폭 병목 × 공속 × 공격/방어 → 피해 → **치명율로 전사/부상 분배** → 병력↓(좌우→중앙 수축).
3. 접전 지속 → 양측 **사기 하락**. 궁은 **정지 시** 사거리 안 전원 사격(이동사격 없음).
4. **종료 = 한쪽 사기 0.** 패배 유닛 **부상병 데리고 도주** → **화면 끝까지 도달**(그동안 피격 시 속수무책 사상자, **추격·퇴로 자동**). **도주 페이즈 ≈10초 상한**, 초과 시 즉시 종료.
5. **장수 생존(도주 종료 시 HP):** 50%+ → **'대기'**(재임명 가능) / 50% 미만 → **부상·사망**(전략 기용 불가). → 승/패 + **정도** 산출. → [03 3.6.1](./03-strategy-layer.md)

### 언덕/고지 효과 (확정 2026-07-03)
**고지 → 저지**(높은 곳에서 낮은 곳으로) 보정:
- **공격력 +20%** (하향 공격)
- **이동력 +30%** (내리막) — 기병은 이 가속으로 **최대속도·charge를 쉽게** 터뜨린다.
- **궁병 사거리 +30%** (고지 사격)
- (매트릭스 궁 언덕 ◎ 반영 → [07 7.3](./07-entities-data.md))

### 지형 템플릿 3종 (초안 2026-07-03)
**랜덤 생성 → "말이 되게" 손질 → 저장 후 재활용**([07 7.3](./07-entities-data.md) 혼합 규칙 대표지형 ≥60%). v0용 3종:
- **T1 평야 회전(會戰)** — 평지 대표, 한쪽에 완만한 **언덕 1** + 개천 한 줄. 정면 대회전 + **언덕 선점** 가치.
- **T2 구릉지** — **언덕/고지 다수** + 평지. 고지 쟁탈전, **궁병 유리**(사거리·하향 보정).
- **T3 애로(隘路)** — **숲·협곡으로 폭이 좁아지는 병목** + 평지. **접전 폭 제한** 데모(소수로 다수 저지).
> 세 장이 각각 다른 축을 시험한다: **정면 폭 / 고지 보정 / 병목**. 구체 배치·시드는 생성 단계에서 확정.

### 승리 & 정도 산출 (확정 2026-07-03)
승리 시 **초반 병력 대비 잔존 비율**로 정도를 가른다(부대 초반 = 2200):
- **80% 이상 유지(≥1760) → 대승리**
- **50% 이상 유지(≥1100) → 승리**
- **50% 미만 → 안타까운 승리**
- v0 표기 예: **"대승리 — 잔존 1820/2200 (83%)"**. (전략 연동 시 이 정도 → 전략 유닛 사기 → [03 3.6.1](./03-strategy-layer.md))

### 장수 v0 (최소)
- **근접 교전**(성향 근접 고정), HP 0 → **부대군기 옆 휴식·리젠**([07 7.1](./07-entities-data.md)).
- 버프는 **기본 규칙만**: 근접전투 10초 지속 → 전술 페이즈당 사기 1회↑. (특성·고무 등은 v0.5)

### v0에서 뺀 것
- 협공(2:1), 전술계략(작전P·간파·위보), 이중 시계·`TIME_RATIO`·전략 알림, 경영 연결. (기병은 **포함**으로 변경)

> ✅ v0 결정 정리:
> - ② 언덕 보정 = **공격 +20% / 이동 +30% / 궁 사거리 +30%** (고지→저지) — **확정**
> - ③ 승리 정도 = 잔존 **≥80% 대승리 / ≥50% 승리 / <50% 안타까운 승리** — **확정**

## 4.9 구현 현실성 노트 (1인 개발)

다행히 **시계 분리 + 한 번에 한 교전**은 구현을 단순하게 만든다.

- **상세(개별 병사) 시뮬은 줌인한 교전 하나만.** 나머지는 가벼운 전략 자동 계산 그대로.
  → 모바일에서도 동시 부하가 작다.
- 두 시계는 같은 시뮬을 다른 **timestep 배율**로 도는 것으로 구현 가능.
- 전략 → 전술 **이벤트 버스**(알림)와, 전술 결과의 **전략 상태 인계**가 핵심 연결부. → [05 문서](./05-tech-architecture.md)
