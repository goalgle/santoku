# 04. 전술 레이어 (확대한 교전 안의 전투 · 실시간 · 느린 시계)

> 전략에서 **자동 계산되던 교전**을 확대하면, 그 안으로 들어가 **유저가 직접 개입**하는 화면이 펼쳐진다.
> 장수·개별 병종·지형 연출이 드러나고, 시간은 전략보다 **느리게** 흐른다.

## 4.1 핵심 아이디어: 자동 계산에 손을 넣는다

전략 레이어의 교전은 기본적으로 **자동 계산**된다(능력치·지형·장수·전방·협공 → 사상자/사기).
플레이어가 그 교전을 **확대(zoom-in)** 하면, 자동으로 흘러가던 전투가 **유저가 개입 가능한
전술 전장**으로 전개된다.

```
   축소 ◀───────────────────────────────────▶ 확대
  [전략: 자동 계산]                    [전술: 유저 개입]
 토큰/카운터, 빠른 시계         장수·개별 병종·지형 연출, 느린 시계
```

- **줌 인의 의미** = "이 교전만큼은 내가 직접 지휘해 결과를 바꾸겠다."
- 줌 인하지 않은 교전은 전략 시계 위에서 계속 자동으로 결판난다.

## 4.2 확대하면 보이는 것

전략에서 토큰 하나였던 교전이, 확대하면 살아 움직인다.

| 요소 | 전략(축소) | 전술(확대) |
|------|-----------|-----------|
| 장수 | 깃발/아이콘 | **통솔 중인 장수 본인**이 등장 (위치·상태) |
| 병종 | 부대 심볼 | **창·궁·기 개별 병사 단위**로 구분·배치 |
| 지형 | Hex 타일 보너스 | **연출·상호작용**: 언덕 경사·물가·낙석 등 |
| 전투 | 수치 자동 계산 | 병사 간 근접/사격이 눈에 보이는 형태로 |

## 4.3 시간 모델: 전략(빠름) vs 전술(느림) — 시계 비율

핵심 규칙. 두 레이어의 시간은 **함께 흐르지만 같지 않다.**

- **전략 시계는 빠르게**, **전술 시계는 느리게** 흐른다.
- 비율은 **튜닝 값**이다. 예: 전술 1초 동안 전략은 N초 진행 (가칭 `TIME_RATIO`, 예 N≈3~5).
- 줌 인하면 → 그 교전만 전술 시계(느림)로 상세 전개. **나머지 전장은 전략 시계(빠름)로 계속 자동 진행.**
- **액티브 포즈로 멈추면 양쪽 모두 정지.** (정지 중 줌을 오가며 명령 예약 가능)

```
플레이어 체감 시간 ──────────────────────────────▶
  전술(줌인한 교전):   천천히, 한 수 한 수 개입 가능
  전략(나머지 전장):   그 사이 빠르게 진행 → 상황이 바뀐다
```

> **설계 의도 = "주의의 안개(fog of attention)".** 한 곳을 깊게 보는 동안 다른 곳에서 일이 벌어진다.
> 시간 비율이 그 대가를 만든다 — 전술에 오래 머물수록 전략 상황은 더 많이 변해 있다.

### 왜 시계를 분리하나 (공유하지 않는 이유)
- **공유(1:1)하면:** 상세 전술 개입을 하는 사이 전장이 거의 멈춘 듯 보여 긴장감이 없다.
- **분리하면:** 전술 개입은 충분히 느려 "둘 만하고", 전략은 빠르게 흘러 "방치의 대가"가 생긴다.
- 이 비대칭이 **선택의 무게**(어디에 개입할까)를 만든다.

## 4.4 전략 상황 공유 (전술 중 알림)

전술에 몰입한 동안 플레이어가 미처 못 본 **전략적 상황이 전술 화면으로 공유**된다.

- 형태: 화면 가장자리 **알림/토스트**, 미니맵 점멸, 경고음(선택).
- 예시:
  - "아군 **A 부대 사기 저하 → 후퇴 중**"
  - "적 증원 부대 접근 (3 Hex)"
  - "B 거점 측면 붕괴 위기"
- 플레이어의 선택지:
  1. **무시하고** 현재 전술 교전에 집중 (이 교전을 빨리 끝낸다)
  2. **액티브 포즈** → 상황 파악 후 명령 예약
  3. **줌 아웃**해 전략적으로 대응 (다른 교전으로 줌인 전환 포함)

> 알림은 "전지적 시야"를 주지 않는다. **무엇을, 언제 알려줄지**가 곧 난이도/긴장 설계. (→ 4.7 OPEN)

## 4.5 전술 명령 (전략과 다른 명령어)

확대 상태에서는 전략의 부대 단위 명령과 **다른, 더 세밀한 명령어**를 쓴다. (방향성 제안)

| 분류 | 예시 명령 (제안) | 설명 |
|------|------------------|------|
| 장수 운용 | 장수 돌격 / 후방 보호 / 특수기 발동 | 장수 본인의 위치·행동 지시 |
| 병종 제어 | 궁병 일제사격 / 창병 방진 / 기병 측면 우회 | 부대 내부 병종별 개별 지시 |
| 지형 활용 | 고지 점령 / 도하 / **낙석 트리거** / 매복 | 전술 지형과의 상호작용 |
| 대형 미세조정 | 산개 / 밀집 / 방향 전환 | 전략의 큰 진형보다 정밀한 배치 |

> **전략 = "어디로 가서 누구와 싸워라"**, **전술 = "그 싸움을 어떻게 이겨라."**
> 같은 액티브 포즈 위에서 명령의 **해상도**가 다르다.

## 4.6 줌 인/아웃 & 교전 전환

- 동시에 깊게 개입할 수 있는 교전은 **한 번에 하나**(권장). 줌인 대상을 바꾸면 이전 교전은 다시 자동 계산.
- 줌 아웃 시: 전술에서의 개입 결과(현재 사상자·사기·배치)를 **그 상태 그대로 전략 자동 계산에 인계**.
- 줌 인 시점: 진행 중인 자동 계산의 **현재 상태에서 이어받아** 전술로 펼친다(처음부터 다시 X).

## 4.7 세부 컨셉에서 정할 것 (OPEN — 함께 결정)

1. **시계 비율(`TIME_RATIO`)** — 전술:전략 = 1:N의 N 값. 체감·밸런스 핵심.
2. **전술 진입 단위** — 교전 1건(부대 쌍)만? 인접한 협공 전체(아군 2 + 적 1)를 한 화면에?
3. **전술 명령의 최종 셋** — 4.5 중 채택/추가.
4. **지형 상호작용 범위** — 낙석 같은 발동형 지형을 어디까지(재미 vs 구현비용).
5. **장수 역할** — 버프형인가, 일기토 같은 개별 전투 주체인가.
6. **알림 정책** — 어떤 전략 이벤트를, 어느 시점에, 얼마나 보여줄지(주의의 안개 난이도).
7. **줌 전환 중 시간** — 전환 순간 자동 정지할지, 완전 실시간 유지할지.

## 4.8 구현 현실성 노트 (1인 개발)

다행히 **시계 분리 + 한 번에 한 교전**은 구현을 단순하게 만든다.

- **상세(개별 병사) 시뮬은 줌인한 교전 하나만.** 나머지는 가벼운 전략 자동 계산 그대로.
  → 모바일에서도 동시 부하가 작다.
- 두 시계는 같은 시뮬을 다른 **timestep 배율**로 도는 것으로 구현 가능.
- 전략 → 전술 **이벤트 버스**(알림)와, 전술 결과의 **전략 상태 인계**가 핵심 연결부. → [05 문서](./05-tech-architecture.md)
