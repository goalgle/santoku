# 01. 게임 개요 (비전 · 컨셉)

## 1.1 한 줄 소개

> 삼국지 군주가 되어 영토를 경영하고, 전장에서는 실시간으로 부대를 지휘하는 토탈워류 전략 게임.
> 단, **언제든 멈춰서 깊게 생각할 수 있다.**

## 1.2 비전 (게임 경험의 핵심)

플레이어는 **하나의 연속된 줌 축** 위에서 세 개의 시점을 오간다.

- **경영의 시점** (턴제, 전체 지도) — 도시를 키우고, 군대를 모으고, 외교를 벌인다.
- **전략의 시점** (실시간, 큰 전장) — 부대들이 보이는 전장에서 부대 단위로 지휘한다.
- **전술의 시점** (실시간, 유닛 확대) — 특정 부대 안으로 파고들어 장수·병종·지형을 직접 다룬다.

전투(전략)에서 한 유닛을 **확대하면** 전술이 펼쳐지고, 두 실시간 레이어는 **하나의 시계를 공유**한다.
이 모두를 잇는 철학이 **액티브 포즈(Active Pause)** 다. 시간은 실시간으로 흐르지만,
플레이어가 "생각하기 위해" 개입하는 순간 세계는 멈춘다.
반사신경이 아니라 **판단력**으로 이기는 게임을 지향한다. 이는 모바일 환경(작은 화면, 터치,
짧은 플레이 세션)과도 잘 맞는다.

## 1.3 핵심 기둥 (Design Pillars)

기획 결정이 흔들릴 때 돌아올 기준점.

1. **판단 > 반응** — 모든 긴박함은 "무엇을 명령할까"의 고민에서 나온다. 손가락 속도가 아니다.
2. **하나의 줌, 끊김 없는 전환** — 전략(부대)에서 전술(병종)로 매끄럽게 파고든다. 별개의 화면이 아니라 같은 전투의 두 배율이다.
3. **실시간 레이어는 한 철학** — 전략·전술은 같은 시계, 같은 액티브 포즈 규칙. 학습 비용을 낮춘다.
4. **모바일에서 진짜 재미있게** — PC 게임의 축소판이 아니라, 터치와 짧은 세션에 맞게 재설계한다.
5. **작게 시작해 깊게** — 1인 개발. 시스템 수는 적게, 각 시스템의 의사결정 깊이는 깊게.

## 1.4 타겟 플레이어

- 토탈워 / 삼국지(코에이) / 문명 류를 좋아하지만, PC 앞에 오래 앉기 어려운 사람
- 전략 게임을 좋아하지만 실시간 컨트롤의 압박은 부담스러운 사람
- 출퇴근·짧은 휴식 시간에 "한 수 두는" 경험을 원하는 모바일 유저

## 1.5 레퍼런스와 차별점

| 레퍼런스 | 가져올 것 | 버리거나 바꿀 것 |
|----------|-----------|------------------|
| Total War: Three Kingdoms | 전략맵 ↔ 전술전투 구조, 삼국지 무드 | 무거운 조작, 고사양, 긴 세션 |
| 코에이 삼국지 | 내정·인재·외교의 깊이 | 복잡한 UI, 학습 곡선 |
| Crusader Kings / Stellaris | 액티브 포즈, 실시간 시뮬 | 방대한 시스템 수 |
| 모바일 SLG | 터치 UX, 짧은 세션 설계 | 과금 중심 설계, 자동 전투 |

**핵심 차별점:** "실시간이지만 멈출 수 있는" 토탈워를, **전략↔전술 연속 줌**으로 한 화면에 잇고,
처음부터 모바일 웹에 맞게 설계.
