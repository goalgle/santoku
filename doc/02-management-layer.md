# 02. 경영 레이어 (턴제 · 전체 지도)

> 군주로서 나라를 다스리는 최상위 레이어. **턴제**로 진행되며, 전체 지도를 내려다본다.
> 내정·외교·군사 편성·부대 이동을 결정한다.

## 2.1 시간 모델: 턴제

경영 레이어는 **턴제**다. 플레이어가 한 턴 동안 명령을 내리고 "턴 종료"를 누르면,
모든 세력의 행동·자원 정산·부대 이동이 일괄 처리된다.

- 1인 개발에 유리: 구현·밸런싱이 단순하고 결정적(deterministic).
- 충분한 사고 시간: 큰 그림(내정·외교)은 느긋하게 결정.
- 실시간 전투(전략/전술)와 명확히 대비 → 페이싱에 리듬을 만든다.

> **레이어 간 시간 관계:** 경영(턴제) → 부대 접촉 발생 → **전략(실시간) 전투 진입** →
> 전투 종료 후 결과를 안고 경영으로 복귀. 자세한 흐름은 [03 문서](./03-strategy-layer.md).

## 2.2 맵 구조

- **노드 기반 맵**으로 시작 (도시·관문을 점, 도로를 선으로 연결).
  - 자유 지형 맵보다 구현이 가볍고, 모바일 터치 타게팅에 유리.
- 각 노드 = **도시/거점**. 노드 간 이동은 도로(엣지)를 따라 N턴 소요.
- 거점 유형: 대도시(생산 거점), 관문(방어 요충), 자원지(특수 자원).

## 2.3 자원

최소 집합으로 시작한다 (1인 개발 — 적게, 깊게).

| 자원 | 용도 | 산출 |
|------|------|------|
| 식량 (糧) | 군대 유지, 인구 성장 | 농지, 도시 |
| 자금 (金) | 건설, 모병, 외교 | 시장, 세금 |
| 병력 (兵) | 부대 편성의 재료 | 인구 + 모병 |
| 민심 (民心) | 반란 억제, 생산 보너스 | 내정 정책 |

> 자원을 4종으로 제한. 각 자원이 서로를 압박하는 **트레이드오프**(예: 세금↑ → 자금↑ 민심↓)를
> 만들어 의사결정에 무게를 둔다.

## 2.4 핵심 시스템

### 내정 (Domestic)
- 도시별 건설 슬롯 → 농지/시장/병영/성벽 등 건물.
- 정책 결정 (세율, 징병 강도) → 자원과 민심의 트레이드오프.

### 군사 (Military)
- 도시의 병력 + 자금으로 **부대(군단)** 편성. 부대는 여러 유닛(보병/궁병/기병)으로 구성.
- 부대는 매 턴 맵을 이동하고, 적 부대/도시와 만나면 **전투 발생 → 전략 레이어로 전환.**
- 여기서 편성한 부대가 곧 전략·전술 레이어에서 지휘할 단위가 된다.

### 인재 (Characters) — 삼국지의 핵심 재미
- 장수는 부대를 지휘하거나 도시를 다스린다. 능력치(무력/지력/통솔/정치).
- **부대에 배속된 장수는 전술 레이어에서 직접 등장**한다 (확대 시 화면에 표시). → [04 문서](./04-tactics-layer.md)
- 등용·배치가 곧 전략. (프로토타입에서는 소수의 핵심 장수만)

### 외교 (Diplomacy)
- AI 세력과 동맹/불가침/선전포고. 프로토타입에서는 최소 형태(전쟁/평화)로 시작.

## 2.5 경영 → 전투 전환

```
[경영맵(턴제): 부대 이동]
      │  적과 접촉 (턴 처리 중)
      ▼
[전투 발생] ──→ "직접 지휘" ──→ [전략 레이어(실시간) / 03 문서]
      │                          │  └→ 유닛 확대 → [전술 / 04 문서]
      └──→ "자동 해결" ←──────────┘ 결과만 반영(빠른 진행)
```

- 모든 전투를 직접 할 필요는 없다. **자동 해결(auto-resolve)** 로 소모적 전투를 건너뛴다.
- 중요한 전투만 실시간(전략/전술)으로 내려가 직접 지휘 → 모바일 세션 길이를 조절.
- **전략계략**은 여기(경영)에서 **전투 시작 전 선택·부여**받는다. 전략 화면에서 유저가 타이밍을 잡아 사용. → [04 4.5.1](./04-tactics-layer.md)

- 인구/성장 모델을 어디까지 시뮬레이션할 것인가?
- AI 세력의 의사결정 깊이 (단순 규칙 기반부터 시작 권장)
- 시나리오: 정통 삼국지 vs 가상 군웅할거? (→ [01 문서](./01-game-overview.md) 열린 질문)
