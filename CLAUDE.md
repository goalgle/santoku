## 프로젝트 한 줄 요약

삼국지 배경의 **토탈워류 전략 게임**. 웹 기반(모바일 우선 → PC), **1인 소규모 프로토타입**.
현재 단계: **기획 문서 작성** (구현 전).

## 핵심 설계 (확정된 것)

- **3개 레이어 + 연속 줌**
  - **경영**: 턴제, 전체 지도. 내정·외교·부대 편성·이동.
  - **전략**: 실시간, **Hex 전장**. 부대 단위 지휘. **교전은 자동 계산**(능력치·병종상성·전방·지형·장수·협공).
  - **전술**: 실시간, 특정 교전을 **확대**해 유저가 직접 개입. 장수·개별 병종·지형 연출이 보임.
- **핵심 메커니즘**
  - **액티브 포즈**: 유저가 개입(명령)하는 순간 전체 정지. 판단력 > 반응속도.
  - **이중 시계**: 전략은 빠르게, 전술은 느리게 흐름(`TIME_RATIO`). 둘은 공유 X, 비율로 분리.
  - **주의의 안개**: 전술 몰입 중 벌어진 전략 상황을 알림으로 공유(예: "A부대 후퇴 중").
- **유닛**: Hex 6면 + **전방(facing)**, 한 타일 한 유닛. 병종 삼각상성(창>기>궁>창). **사기 붕괴**로 결판.
- **기술 방향(미확정)**: PixiJS + TypeScript + Vite, 클라이언트 온리. 줌인 교전 1건만 정밀 시뮬.

### 확정 요약 (장수 시스템 — 2026-06-30~07-01)
- **능력치 6종**: 통·무·지·정·매·운 (0~100). `통솔→지휘반경`, `지력→계략범위` 대칭.
- **운=시드 RNG**(결정성 유지, `doc/05`). 매력·정치는 프로토타입 **데이터만**(미사용).
- **장수 = 버프+일기토 하이브리드**: HP 보유, 출진/휴식, **부대군기**로 리젠.
- **부대군기**: 이동=보병 1/2. **부서지지 않으면** ①명령(반경=통솔 비례) ②HP회복·리젠 포인트.
  **부서지면** 명령·리젠 불가 + 사기↓.
- **도주·재편성**: `사기0 & 장수HP0` → 군기 유무 무관 **도주** → **대장군기 집결**(소멸 아님) →
  일정 수+여분 장수 시 **저사기 유닛 재편성**. ("해산" 용어 폐기)
- **계략**(=장수 특수기, 커맨드와 구분): 제자리 작전P 차징→피격 시 취소, 범위=지휘반경+지력.
- **전술계략 간파**: 방어측 지략 주사위(대성공=종류+타겟/성공=종류/실패). **간파·위보는 장수 특기**로
  차징 시 주사위 ±보정(간파=+, 위보=상대−). **거짓정보 없음**(복잡도로 미도입).
- **사기**(`doc/03` 3.6.1): 0~100 시작 50, 증감 요인 표, 데미지 작용, ≤20 명령 반응 실패.
- **장수 특성 = 패시브**(조건 발동, 기본 규칙 완화형; `doc/07` 7.1). 후보 5종:
  고무(근접전투 지속→사기, 10→7초·2회) / 간파·위보(계략 간파 주사위 ±) / 추격·퇴로(도주).
  기본 규칙: 장수 **근접전투 10초 지속 → 전술 페이즈당 사기 1회↑**.

### 확정 요약 (전술 표현·지형 — 2026-07-01)
- **전술 = 연속 공간(Hex 아님) + 대형 덩어리**(`doc/04` 4.2.1): 장수=단일 에이전트,
  병종=그룹 단위 시뮬(앵커+대형 슬롯 스프라이트+노이즈). facing=연속 각도.
- **레이어 격리 & 최소 인터페이스**(`doc/04` 4.6, `doc/05` 5.4): 전술은 독립 레이어.
  **진입=전략의 교전 스냅샷 / 복귀=승·패+승리의 정도**(정도→전략 유닛 사기). 세부 상태 되돌림·위치 변환 **없음**.
- **협공 & 동적 구성**(`doc/04` 4.6): 전술은 1:1만 아님. **2:1 협공**은 Hex 인접 방향대로 진입.
  전술 진행 중 전략 실시간 연동으로 **구성 변동(1:1↔2:1)**. 시간은 `TIME_RATIO` 공유+알림.
- **전략 유닛 사기 원천**(`doc/03` 3.6.1): 보급/후퇴·지원 소식 + **전술 승리의 정도** + **전략 계략 성공**.
- **명령·계략 3종 체계**(`doc/04` 4.5.1): **전략계략**(전략 화면만, 경영서 전투 전 부여, 타이밍 유저) /
  **전술계략**(장수 작전P 차징, 범위=명령반경+지력, 간파·위보 / =기존 '계략') /
  **전술명령**(부대군기 있으면 가능, 장수 없어도 발동; 예 돌격·기마저지·방어전념).
- **지형**(`doc/07` 7.3): 항목화(평지/개천/강/언덕/산악/숲/도하/협곡). **전략→전술 혼합 = 대표 지형 ≥60% + 잔여 혼합**, 시드 절차생성.
- **병종**(`doc/07` 7.2): 한 유닛=여러 병종. **로스터 5종** = 방패병(검병)·창병·궁병·기병·깃발병.
  능력치 8종 = 공격/방어/**병수(HP)**/**부상병(복구 가능 HP)**/이동력/**회전(방향전환)**/사거리/저지력. 사거리 예 궁7·창2·방1.
  저지력=상대 이동력 감소. **피해=전사(영구)+부상병(도주·회복 복구)**.
  능력치는 **등급 S/A/B/C/D(+특수 E)** 로 표기(장수는 0~100 수치). 등급표 제안값(˟) 확정은 OPEN.
  **깃발병 = 부대군기 유닛**(잠정 결정, 확인 요망).

## 작업 규칙 (convention)

- 문서는 **living document** — 결정이 바뀌면 관련 `doc/*`와 이 파일을 함께 갱신.
- 밸런싱 수치는 **코드가 아니라 데이터로** 둔다(JSON/TS 분리).
- 커밋 후 `git push`로 동기화(이게 PC 간 작업 이어가는 통로).
- 문서는 한국어로 작성.

## 다른 PC에서 이어받는 법

1. `git clone https://github.com/goalgle/santoku.git && cd santoku`
2. 그 폴더에서 `claude` 실행 → 이 `CLAUDE.md`와 `doc/`를 읽고 맥락 복원.
3. (선택) 이전 대화 흐름까지 그대로 옮기고 싶으면, 기존 PC의
   `~/.claude/projects/<프로젝트경로>/<세션id>.jsonl`을 새 PC의 동일 경로에 복사 →
   `claude --resume`로 선택. ⚠️ 비공식 방법이며, 절대경로가 같아야 하고 버전에 따라 깨질 수 있음.
   → **권장은 이 파일 기반으로 새 세션을 시작하는 것.**
